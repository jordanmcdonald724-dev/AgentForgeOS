# AgentForgeOS — Studio UI Shell Lock

This document defines the permanent Studio interface shell. The layout below **must remain stable across all UI implementations** to prevent drift when new features are added or generated by agents.

## Required Regions

- **Top Navigation Bar** — Provides global navigation and system controls (project/workspace selectors, status, settings, profile).
- **Left Sidebar (Module Navigation)** — Hosts module and tool navigation; all modules must appear here.
- **Main Workspace** — Primary work area that changes per module while preserving the surrounding shell.
- **Agent Console** — Dedicated area for prompting and viewing agent responses and logs.
- **Pipeline Monitor** — Dedicated area to visualize pipeline stages, active agents, and execution state.

## Stability Rule

These shell regions are permanent and must not be removed or replaced. New UI components must render **inside these regions**, not overwrite or relocate them. Resize, theming, or content changes are allowed only when they preserve the shell structure above.***
